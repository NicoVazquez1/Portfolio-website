  • Agrolo.

    Aplicación móvil para ahorrar tiempo

    Tecnologías:

    Responsabilidades: Diseño del producto | Diseño de la interfaz | Diseño del modelo de base de datos | Backend | Tareas comerciales | Formación de otros miembros

    Estado: Esperando al cliente

    Descripción: Este proyecto tiene como objetivo agilizar el proceso de registro de peso de frutas, ingresos generados, pagos a los trabajadores, cantidad de bandejas, información del cliente y tipo de entrega en un campo de arándanos. Yo mismo trabajé en un campo similar hace algunos años y vi de primera mano lo lento que era el proceso y cómo se podía mejorar.

    No puedo proporcionar el código fuente ya que es un producto para un cliente.

    Anteriormente, el proceso de recolección de frutas implicaba cálculos manuales y registros en papel. Para mejorar la eficiencia, propuse crear una aplicación que automatice el registro y procesamiento de datos.

    El dueño del campo aprobó la idea, y diseñé la aplicación en consecuencia. También creé un diagrama de entidad-relación para visualizar la estructura de datos.

    A continuación, estimé el presupuesto y creé historias de usuario para abordar los problemas identificados.

    Utilizando Entity Framework, establecí la conexión con la base de datos e implementé medidas de seguridad como JWT.

    La aplicación sigue una arquitectura MVC, con puntos finales en la carpeta de servicios y la lógica empresarial manejada por la clase NegocioService.

    Realicé pruebas utilizando Postman, que incluyeron la creación de un nuevo camión y la realización de entregas de frutas.

    El sistema calcula los kilogramos totales entregados restando el peso de las bandejas del peso total. Los pagos se calculan en base al pago por kilogramo y se asignan a los trabajadores.

    Todos los datos se almacenan en la base de datos, y la aplicación permite el manejo adecuado de la carga de frutas y las salidas de camiones.

    Actualmente, el proyecto se encuentra funcionando correctamente en el servidor, y se está brindando asistencia adicional para el desarrollo del frontend. El proyecto aún está en progreso debido a consideraciones financieras, y estoy esperando más requisitos por parte del cliente.

    Magic Cannon

    Un videojuego shooter en 3D

    Tecnología utilizada:

    Responsabilidades: Selección de activos | Lógica de disparo | Movimiento

    Estado: Activo (fase muy temprana de desarrollo)

    Magic Cannon es un juego en 3D donde disparas barriles que aparecen en la pantalla utilizando tu cañón. En el juego puedes disparar 3 tipos de balas para lograr diferentes estrategias.

    GitHub Repositorio

    Creé este videojuego para poner en práctica mis habilidades de programación y aprender el motor de Unity. Comencé con una versión antigua y logré hacer que el cañón disparara objetivos con diferentes tipos de munición, sonidos y efectos de partículas. Sin embargo, a medida que el proyecto comenzó a crecer, se volvió un poco desordenado. Así que comencé una nueva versión utilizando parte del código utilizado anteriormente. Así es cómo se veía el proyecto antes:

    La nueva versión hace que la jugabilidad sea más fluida y mejora el rendimiento.

    Esto es lo que he logrado hasta ahora en 2 días. Puedes cambiar entre tres tipos de munición:

    Bala estándar: una sola bala ideal para la precisión.

    Disparo de escopeta: múltiples balas con dispersión, ideal para golpear rápidamente múltiples objetivos cercanos.

    Bala congelante: munición explosiva ideal para golpear múltiples objetivos de un solo golpe.

    Próximos pasos: implementar un sistema de puntuación, HUD, sistema de recompensas dependiendo de los diferentes barriles, mejorar el rendimiento, mejorar la jugabilidad, corregir errores.

    También, aquí está el código fuente del proyecto. Recuerda, este proyecto solo tiene un par de horas de trabajo, por lo que se encuentra en una fase muy temprana de desarrollo.
