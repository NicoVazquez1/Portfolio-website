  • Agrolo.

    Mobile app

    Stack:

    Responsabilities: Product design | Interface design | Database model design | Backend | Buisness task | Formation of the other member

    State: Waiting for client requiriment

    Description: This project aims to streamline the process of recording fruit weight, revenue generated, worker payments, tray quantity, client information, and delivery type in a blueberry field. I myself worked in such a field a few years ago and witnessed firsthand how slow the process was and how it could be improved.

    I Cant' provide the source code as it is a product for a client

    Previously, the fruit collection process involved manual calculations and paper records. To improve efficiency, I proposed creating an app that automates data recording and processing.

    The field owner approved the idea, and I designed the app accordingly. I also created an entity-relationship diagram to visualize the data structure.

    Next, I estimated the budget and created user stories to address the identified problems.

    Using Entity Framework, I established the database connection and implemented security measures such as JWT.

    The app follows an MVC architecture, with endpoints in the services folder and business logic handled by the NegocioService class.

    I conducted tests using Postman, including creating a new truck and making fruit deliveries.

    The system calculates the total kilograms delivered by subtracting the weight of trays from the total weight. Payments are calculated based on the payment per kilogram and assigned to workers.

    All data is stored in the database, and the app allows for the proper management of the fruit load and truck departures.

    The project is currently running smoothly on the server, and additional assistance is being provided for frontend development. The project is still in progress due to financial considerations, and I am awaiting further requirements from the client.

    From my perspective as a programmer without prior experience, working on the Agrolo mobile app has been a significant achievement for me. Being involved in the entire development process, from product design to database modeling, has allowed me to gain valuable skills and knowledge. One of the biggest accomplishments in this project is identifying a real-world problem in the fruit collection process and proposing a solution that streamlines the workflow. I took the initiative to design the app, create an entity-relationship diagram, estimate the budget, and develop user stories. Implementing features such as database connectivity, security measures, and MVC architecture using technologies like Entity Framework and JWT has been a tremendous learning experience. Conducting tests, ensuring data accuracy, and providing ongoing support for frontend development has further enhanced my skills. Although the project is still in progress due to financial considerations and awaiting further client requirements, I take pride in the fact that the app is running smoothly on the server and contributing to the improved efficiency of fruit collection operations. Overall, this achievement has allowed me to showcase my problem-solving abilities, technical skills, and adaptability as a programmer.

    Flowtest.

    Automation Test Framework

    Stack:

    Responsabilities: Arquitecture | Whole development

    State: Completed

    Description: This is my Automation framework, written in C# and using Selenium and SpecFlow. It has hooks (which are processes that are executed every time a test case, scenario, or step begins or finishes), reporting of the number of bugs, and the state of the project.

    This is the report of the test results

    GitHub Repository

    In the features folder of our project, we create Gherkin files that define the behavior of our tests. Using SpecFlow, we can automatically generate the corresponding methods based on these files.


    Let's take a specific scenario as an example. We start by navigating to YouTube and entering the query "playing god". Once the program reaches the search results page, it clicks on the first video and verifies that the title matches the expected value using an assert function.

    Additionally, the project includes an extended report feature that generates a Report for each test case.

    You can find the project on my GitHub for more details and access to the code.
